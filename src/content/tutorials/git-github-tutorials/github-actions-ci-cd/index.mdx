# GitHub Actions CI/CD: Complete Setup Guide

GitHub Actions is a powerful CI/CD platform that automates your software development workflows. Learn how to set up automated testing, building, and deployment.

## What is GitHub Actions?

GitHub Actions is a continuous integration and continuous deployment (CI/CD) platform that allows you to automate your software development workflows directly in your GitHub repository.

### Key Features

- **Event-driven**: Trigger workflows on push, pull request, release, etc.
- **Matrix builds**: Test across multiple environments
- **Secrets management**: Secure handling of sensitive data
- **Marketplace**: Thousands of pre-built actions
- **Self-hosted runners**: Run workflows on your own infrastructure

## Getting Started

### Basic Workflow Structure

```yaml
# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - name: Checkout code
      uses: actions/checkout@v3
      
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
        
    - name: Install dependencies
      run: npm ci
      
    - name: Run tests
      run: npm test
      
    - name: Run linting
      run: npm run lint
```

### Workflow Triggers

```yaml
# Multiple triggers
on:
  push:
    branches: [ main, develop ]
    paths: [ 'src/**', 'package.json' ]
  pull_request:
    branches: [ main ]
    types: [ opened, synchronize, reopened ]
  release:
    types: [ published ]
  schedule:
    - cron: '0 0 * * 0'  # Weekly on Sunday
  workflow_dispatch:  # Manual trigger
```

## Essential Actions

### Checkout Code

```yaml
- name: Checkout code
  uses: actions/checkout@v3
  with:
    fetch-depth: 0  # Fetch all history
    token: ${{ secrets.GITHUB_TOKEN }}
```

### Setup Node.js

```yaml
- name: Setup Node.js
  uses: actions/setup-node@v3
  with:
    node-version: '18'
    cache: 'npm'
    cache-dependency-path: 'package-lock.json'
```

### Setup Python

```yaml
- name: Setup Python
  uses: actions/setup-python@v4
  with:
    python-version: '3.11'
    cache: 'pip'
```

### Setup Docker

```yaml
- name: Set up Docker Buildx
  uses: docker/setup-buildx-action@v2
  
- name: Login to Docker Hub
  uses: docker/login-action@v2
  with:
    username: ${{ secrets.DOCKER_USERNAME }}
    password: ${{ secrets.DOCKER_PASSWORD }}
    
- name: Build and push
  uses: docker/build-push-action@v4
  with:
    context: .
    push: true
    tags: myapp:latest
```

## Advanced Workflows

### Matrix Builds

```yaml
name: Test Matrix

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node-version: [16, 18, 20]
        os: [ubuntu-latest, windows-latest, macos-latest]
        
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js ${{ matrix.node-version }}
      uses: actions/setup-node@v3
      with:
        node-version: ${{ matrix.node-version }}
        
    - name: Install dependencies
      run: npm ci
      
    - name: Run tests
      run: npm test
```

### Conditional Steps

```yaml
- name: Run tests
  if: github.event_name == 'pull_request'
  run: npm test

- name: Deploy to staging
  if: github.ref == 'refs/heads/develop'
  run: npm run deploy:staging

- name: Deploy to production
  if: github.ref == 'refs/heads/main'
  run: npm run deploy:production
```

### Parallel Jobs

```yaml
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - name: Run tests
      run: npm test
      
  build:
    runs-on: ubuntu-latest
    needs: test
    steps:
    - uses: actions/checkout@v3
    - name: Build application
      run: npm run build
      
  deploy:
    runs-on: ubuntu-latest
    needs: [test, build]
    steps:
    - name: Deploy application
      run: echo "Deploying..."
```


## Deployment Strategies

### Deploy to AWS

```yaml
- name: Configure AWS credentials
  uses: aws-actions/configure-aws-credentials@v2
  with:
    aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
    aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
    aws-region: us-east-1
    
- name: Deploy to AWS
  run: |
    aws s3 sync dist/ s3://my-bucket
    aws cloudfront create-invalidation --distribution-id ${{ secrets.CLOUDFRONT_ID }} --paths "/*"
```

### Deploy to Vercel

```yaml
- name: Deploy to Vercel
  uses: amondnet/vercel-action@v25
  with:
    vercel-token: ${{ secrets.VERCEL_TOKEN }}
    vercel-org-id: ${{ secrets.ORG_ID }}
    vercel-project-id: ${{ secrets.PROJECT_ID }}
    working-directory: ./
```

### Deploy to Kubernetes

```yaml
- name: Deploy to Kubernetes
  uses: azure/k8s-deploy@v1
  with:
    manifests: |
      k8s/deployment.yml
      k8s/service.yml
    images: |
      myapp:${{ github.sha }}
    namespace: production
```

## Caching Strategies

### Node.js Caching

```yaml
- name: Cache node modules
  uses: actions/cache@v3
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}
    restore-keys: |
      ${{ runner.os }}-node-
```

### Docker Layer Caching

```yaml
- name: Build with cache
  uses: docker/build-push-action@v4
  with:
    context: .
    push: true
    tags: myapp:latest
    cache-from: type=gha
    cache-to: type=gha,mode=max
```

### Custom Caching

```yaml
- name: Cache build artifacts
  uses: actions/cache@v3
  with:
    path: |
      dist/
      build/
    key: ${{ runner.os }}-build-${{ hashFiles('**/package-lock.json') }}
```

## Best Practices

### 1. Use Specific Action Versions

```yaml
# Good
- uses: actions/checkout@v3

# Bad
- uses: actions/checkout@master
```

### 2. Optimize Workflow Performance

```yaml
# Use concurrency to prevent duplicate runs
concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true
```

### 3. Clean Up Resources

```yaml
- name: Cleanup
  if: always()
  run: |
    docker system prune -f
    rm -rf node_modules
```

### 4. Use Environment Variables

```yaml
env:
  NODE_ENV: production
  DEBUG: false

jobs:
  build:
    runs-on: ubuntu-latest
    env:
      CUSTOM_VAR: value
    steps:
    - name: Build
      run: npm run build
```

### 5. Implement Proper Error Handling

```yaml
- name: Run tests
  continue-on-error: true
  run: npm test

- name: Check test results
  if: steps.test.outcome == 'failure'
  run: echo "Tests failed but continuing..."
```
